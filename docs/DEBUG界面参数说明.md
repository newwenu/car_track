# DEBUG 界面显示参数说明

## 进入与退出

- **进入**：在初始等待状态（`FSM_STATE_IDLE`）下，长按 **模式键 2 秒**，从主界面进入 DEBUG 页。
- **切换页面**：在 DEBUG 页中，**短按模式键** 循环切换以下 6 个子页面：INFO → MIC → MOTOR → USONIC → TRACE → ACT。
- **退出**：在 DEBUG 页中，**再次长按模式键 2 秒**，退出 DEBUG 并返回主界面；退出时会自动停止所有调试输出（电机、报警 LED、蜂鸣器）。

> 注意：电机自检页仅在 `IDLE` 状态有效；若 FSM 切到运行/避障/刹车/完成状态，该页会自动跳过电机动作。

## OLED 页面参数说明

### 1. INFO — 综合内部变量

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `E` | 当前循迹误差 × 100 | `trace_control_get_error() * 100`，范围约 `-200 ~ +200` |
| `D` | 超声波测距结果 | `obstacle_guard_get_distance()`，单位 cm；`--` 表示无效/超量程 |
| `V` | 当前车速 | `vehicle_get_speed_cm_s()`，单位 cm/s |
| `L` | 当前已跑圈数 / 目标圈数 | `lap_counter_get_laps()`，目标固定显示为 2 |
| `TL / TR` | 左右电机目标速度百分比 | `motion_get_target()` |
| `ML / MR` | 左右电机当前速度百分比 | `motion_get_current()` |
| `FSM` | 当前系统状态机状态 | `fsm_get_state()` |
| `ENC` | 左右编码器累计脉冲数（不清零） | `encoder_get_counts()`，上电以来的累计总量 |

> `ENC` 是 `encoder_get_counts()` 返回的**上电以来累计总量**，不会被 `encoder_read()` 的清零动作影响；而车速/里程模块每 100ms 调用 `encoder_read()` 取走的是期间增量。若需要清零累计总量，需复位单片机。

### 2. MIC — 声控咪头检测与拍手识别

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `MIC:ON/OFF` | 声控功能使能状态 | `mic_is_enabled()`；`ON` 启用，`OFF` 禁用 |
| `*` | 电平变化指示 | 当前原始电平与上次不同 |
| `RAW` | 当前原始引脚电平 | `mic_get_raw()`；`0` 显示为 `LOW`，非 0 显示为 `HI` |
| `TRG` | 触发计数 | 进入页面后检测到的有效拍手次数 |
| `DEB` / `WIN` | 消抖参数 | 3 次连续采样（约 30 ms）为有效触发；触发窗口约 800 ms |

操作说明：

- **KEY_START（模式键）短按**：切换声控使能 / 禁用。
  - 启用：蜂鸣器 1200 Hz 短鸣提示。
  - 禁用：蜂鸣器 800 Hz 短鸣提示。
- **KEY_EXT 短按**：复位触发计数器。
- 使能后拍手，触发计数增加并显示距离上次触发的时长。

### 3. MOTOR — 电机直线前进与补偿标定

| 显示 | 含义 | 说明 |
|------|------|------|
| `MOT:FWD x%` | 当前电机方向 / 速度 | 直线前进，速度可调，默认 80% |
| `MOT:IDLE S:x` | 电机未运行 | 当前 FSM 不是 `IDLE` 状态，无法自检 |
| `L / R` | 标定结束时左右编码器脉冲数 | 固定速度直行 2 秒后采集 |
| `LG / RG` | 建议左右电机补偿增益 | 根据编码器差值自动计算并应用 |
| `GAIN Lx.xx Rx.xx` | 当前实际补偿增益 | `motion_get_gains()` 返回值 |

操作说明：

- **KEY_START（模式键）短按**：在非标定状态下循环切换前进速度档位（步进 5%，范围 20%~100%）。
- **KEY_EXT 短按**：触发/重新触发 **电机补偿标定**。
  - 标定中再次短按：提前结束并计算结果。
- 标定过程：小车以固定速度 **60%** 直行 **2 秒**，期间禁用已有补偿；结束后根据左右编码器脉冲差自动计算并应用建议增益。

> 电机自检只在 `IDLE` 状态有效；退出 DEBUG 或切到其他页面会自动停车。自检时电机会真实转动，建议架空车轮或按住小车，防止掉落。

### 4. USONIC — 超声波原始数据与诊断

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `D` | 障碍距离 cm | `obstacle_guard_get_distance()`；`--` 表示超量程 |
| `ECHO` | 原始回波时间 | `ultrasonic_get_raw()`，单位 μs；`TOUT(999)` 表示超时 |
| `OK` | 测距完成标志 | 一次测量是否成功返回 |
| `LSTN` | 监听回波标志 | 是否正在等待回波 |
| `PIN` | ECHO 引脚电平 | `H` 高电平 / `L` 低电平 |
| `RAW` | 直接读取的超声距离 | `ultrasonic_get_distance()`，EMA 滤波后的值；`>99cm` 表示远方 |
| `TOUT_CNT` | 连续超时计数 | `ultrasonic_get_timeout_cnt()`，用于诊断探头/接线 |

### 5. TRACE — 循迹原始数据

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `A0 ~ A3` | 4 路循迹传感器原始 ADC 值 | `trace_read(adc)`；大于 `2000` 通常认为在黑线上 |
| `E` | 当前循迹误差 × 100 | 同 INFO 页 |
| `LE` | 上一次有效循迹误差 × 100 | `trace_control_get_last_error() * 100`，用于观察 D 项和丢线惯性 |
| `LC` | 连续丢线计数 | `trace_control_get_lost_info()` |
| `RC` | 丢线后寻线恢复计数 | `trace_control_get_lost_info()` |

> 刚通电且传感器悬空/未对黑线时，4 路 ADC 基线应接近（差异由硬件容差、环境光、安装高度等造成）。若某一路明显偏离，需检查对应传感器、电阻或安装高度。

### 6. ACT — 外部报警 LED + 蜂鸣器自检

| 显示 | 含义 | 说明 |
|------|------|------|
| `ACT:RUN T:xxx` | 自检运行中 | 外部报警 LED 和蜂鸣器按周期翻转/鸣叫 |
| `ACT:SKIP S:x` | 当前状态不允许自检 | 仅在 `IDLE` 或 `FINISHED` 状态才运行 |

> 该页只操作 **外部报警 LED（LED_ALARM）和蜂鸣器**，不涉及内部状态指示灯和按键。

## 串口输出格式

DEBUG 页激活后，每隔 **500 ms** 通过串口输出两行数据（波特率 9600），方便 PC 端查看或录波。

### 第一行 — 综合运行数据

```text
[DBG:INFO] E:+12 D: 45 V:30 L:1 A:2100,1900,1800,1950,2200 TL:+35 TR:+35 ML:+34 MR:+35 FSM:2 MOT:1 LOST:0
```

| 字段 | 说明 |
|------|------|
| `DBG:INFO` | 当前 DEBUG 子页面名称（INFO / MIC / MOTOR / USONIC / TRACE / ACT） |
| `E` | 循迹误差 × 100 |
| `D` | 障碍距离 cm，`--` 表示无效 |
| `V` | 车速 cm/s |
| `L` | 已跑圈数 |
| `A0~A3` | 4 路 ADC，逗号分隔 |
| `TL / TR` | 左右目标速度 |
| `ML / MR` | 左右当前速度 |
| `FSM` | 状态机状态 |
| `MOT` | 电机控制状态 |
| `LOST` | 丢线标志 |

### 第二行 — 新增内部状态

```text
[DBG:INFO] EL:  1234 ER:  1235 LE:  +12 LC:0 RC:0 US: 350 O:1 I:1
```

| 字段 | 说明 |
|------|------|
| `EL / ER` | 左右编码器累计脉冲 |
| `LE` | 上一次有效循迹误差 × 100 |
| `LC` | 连续丢线计数 |
| `RC` | 寻线恢复计数 |
| `US` | 超声波原始回波时间 μs |
| `O` | 测距完成标志 |
| `I` | 监听回波标志 |

## 安全说明

1. DEBUG 页通过 `ui_debug_is_active()` 开关控制；未进入 DEBUG 页时，`ui_debug_update()` 不会执行任何动作。
2. 所有 IO 自检（电机、LED、蜂鸣器）仅在 `IDLE` 或 `FINISHED` 状态有效，避免影响正常行驶。
3. 退出 DEBUG 页时会调用 `ui_debug_stop_all()`，强制关闭报警 LED、蜂鸣器并停止电机。
4. 串口打印仅在 DEBUG 页激活时输出，不会在平时运行中产生额外数据。
