# DEBUG 界面显示参数说明

## 进入与退出

- **进入**：在初始等待状态（`FSM_STATE_IDLE`）下，长按 **模式键（KEY_START，PA5）2 秒**，从主界面进入 DEBUG 页。
- **切换页面**：在 DEBUG 页中，**短按模式键** 循环切换以下 **7** 个子页面：INFO → MIC → TRACE → ACT → USONIC → MOTOR → ENCODER。
- **退出**：在 DEBUG 页中，**再次长按模式键 2 秒**，退出 DEBUG 并返回主界面；退出时会自动停止所有调试输出（电机、报警 LED、蜂鸣器）。

> 注意：电机自检页（MOTOR）和 ACT 自检页仅在 `IDLE` 状态有效；若 FSM 切到运行/避障/刹车/完成状态，这些页面会自动跳过相关动作。

## OLED 页面参数说明

### 1. INFO — 综合内部变量

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `E` | 当前循迹误差 × 100 | `trace_control_get_error() * 100`，4 路传感器误差范围约 `-150 ~ +150` |
| `D` | 超声波测距结果 | `obstacle_guard_get_distance()`，单位 cm；`-1` 表示 ≥50cm（远方/无效） |
| `V` | 当前车速 | `vehicle_get_speed_cm_s()`，单位 cm/s |
| `L` | 当前已跑圈数 / 目标圈数 | `lap_counter_get_laps()`，目标固定显示为 2 |
| `TL / TR` | 左右电机目标速度百分比 | `motion_get_target()` |
| `ML / MR` | 左右电机当前速度百分比 | `motion_get_current()` |
| `FSM` | 当前系统状态机状态 | `fsm_get_state()`：0=IDLE, 1=RUNNING, 2=AVOIDING, 3=BRAKING, 4=FINISHED |
| `ENC` | 左右编码器累计脉冲数（不清零） | `encoder_get_counts()`，上电以来的累计总量 |

> `ENC` 是 `encoder_get_counts()` 返回的**上电以来累计总量**，不会被 `encoder_read()` 的清零动作影响；而车速/里程模块每 100ms 调用 `encoder_read()` 取走的是期间增量。若需要清零累计总量，需复位单片机。

### 2. MIC — 声控咪头检测与拍手识别

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `MIC:ON/OFF` | 声控功能使能状态 | `mic_is_enabled()`；`ON` 启用，`OFF` 禁用 |
| `*` | 电平变化指示 | 当前原始电平与上次不同时显示 |
| `RAW` | 当前原始引脚电平 | `mic_get_raw()`；`0`=LOW（有声），`1`=HI（无声） |
| `TRG` | 触发计数 | 进入页面后检测到的有效拍手次数 |
| `S/D/-` | 最近触发类型 | S=单次拍手, D=双次拍手, -=无 |
| `DEB` / `WIN` | 消抖参数 | 3 次连续采样（约 30 ms）为有效触发；双击窗口约 800 ms |

操作说明：

- **KEY_START（模式键）短按**：切换声控使能 / 禁用。
  - 启用：蜂鸣器 1200 Hz 短鸣提示。
  - 禁用：蜂鸣器 800 Hz 短鸣提示。
- **KEY_EXT（外接键）短按**：复位触发计数器。
- 使能后拍手，触发计数增加并显示距离上次触发的时长。

> 注意：声控功能默认关闭。启用后在 IDLE/FINISHED 状态单次拍手启动小车，运行中连续两次拍手紧急停车复位。触发检测由 `app.c` 10ms 循环中的 `mic_scan()` 执行，不在此页面内主动扫描。

### 3. TRACE — 循迹原始数据

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `A0 ~ A3` | 4 路循迹传感器原始 ADC 值 | `trace_read(adc)`；AO1~AO4 对应 PA2/PA3/PA6/PA7 |
| `E` | 当前循迹误差 × 100 | 同 INFO 页，范围约 `-150 ~ +150` |
| `LE` | 上一次有效循迹误差 × 100 | `trace_control_get_last_error() * 100`，用于观察 D 项和丢线惯性 |
| `LC` | 连续丢线计数 | `trace_control_get_lost_info()`，≥3（60ms）进入丢线状态 |
| `RC` | 丢线后寻线恢复计数 | `trace_control_get_lost_info()`，超时 150（3s）后停车 |

> **黑线判定**：当前使用**施密特触发器**，ADC > 750 判定变黑（上升阈值），ADC < 600 判定变白（下降阈值），死区 150 抑制黑白边界抖动。不再使用单一 `TRACE_BLACK_THR=2000` 阈值。

> 刚通电且传感器悬空/未对黑线时，4 路 ADC 基线应接近（差异由硬件容差、环境光、安装高度等造成）。若某一路明显偏离，需检查对应传感器、电阻或安装高度。

### 4. ACT — 外部报警 LED + 蜂鸣器自检

| 显示 | 含义 | 说明 |
|------|------|------|
| `ACT:RUN T:xxx` | 自检运行中 | 外部报警 LED 和蜂鸣器按周期翻转/鸣叫 |
| `ACT:SKIP S:x` | 当前状态不允许自检 | 仅在 `IDLE` 或 `FINISHED` 状态才运行 |

自检行为：
- LED（PB10）：每 500ms 翻转一次
- 蜂鸣器（PB1）：每 1000ms 短鸣一次（1000Hz, 100ms）

> 该页只操作 **外部报警 LED（LED_ALARM）和蜂鸣器**，不涉及内部状态指示灯（PA4）和按键。

### 5. USONIC — 超声波原始数据与诊断

| 显示 | 含义 | 来源/说明 |
|------|------|-----------|
| `D` | 障碍距离 cm | `obstacle_guard_get_distance()`；`--` 表示 ≥50cm（远方/超时） |
| `ECHO` | 原始回波时间 | `ultrasonic_get_raw()`，单位 μs；`TOUT(999)` 表示超时（值 0xFFFF） |
| `OK` | 测距完成标志 | 一次测量是否成功返回（1=完成） |
| `LSTN` | 监听回波标志 | 是否正在等待回波（1=等待中） |
| `PIN` | ECHO 引脚电平 | `H` 高电平 / `L` 低电平 |
| `RAW` | EMA 滤波后的超声距离 | `ultrasonic_get_distance()`；`>99cm` 表示远方 |
| `TOUT_CNT` | 连续超时计数 | `ultrasonic_get_timeout_cnt()`，用于诊断探头/接线 |

> **超声波参数**：TIM4 分辨率 1μs（PSC=71, ARR=4999），测距范围约 4~86cm。距离公式为 `(250μs盲区 + 回波时间) / 58.0` cm。EMA 自适应滤波（α=0.3/0.5/0.8）平滑输出。超时分三阶段处理（30ms 极短记忆 → 80ms 快速退出 → 明确远距）。

### 6. MOTOR — 电机直线前进与补偿标定

| 显示 | 含义 | 说明 |
|------|------|------|
| `MOT:FWD x%` | 当前电机方向 / 速度 | 直线前进，速度可调，默认 80% |
| `MOT:IDLE S:x` | 电机未运行 | 当前 FSM 不是 `IDLE` 状态，无法自检 |
| `L / R` | 标定结束时左右编码器脉冲数 | 固定速度直行 2 秒后采集 |
| `LG / RG` | 建议左右电机补偿增益 | 根据编码器差值自动计算并应用 |
| `GAIN Lx.xx Rx.xx` | 当前实际补偿增益 | `motion_get_gains()` 返回值 |

操作说明：

- **KEY_START（模式键）短按**：在非标定状态下循环切换前进速度档位（步进 5%，范围 20%~100%）。
- **KEY_EXT（外接键）短按**：触发/重新触发 **电机补偿标定**。
  - 标定中再次短按：提前结束并计算结果。
- 标定过程：小车以固定速度 **60%** 直行 **2 秒**，期间**临时禁用已有补偿**（增益重置为 1.0/1.0）；结束后根据左右编码器脉冲差自动计算并**应用建议增益**（降低较快一侧）。

> 电机自检只在 `IDLE` 状态有效；退出 DEBUG 或切到其他页面会自动停车。自检时电机会真实转动，建议架空车轮或按住小车，防止掉落。标定结果会自动更新到 `motion_control.c` 中的增益变量，但**断电不保存**，需手动将 OLED 上显示的 LG/RG 值写入代码重新编译。

### 7. ENCODER — 编码器 PPR 实测（新增）

| 显示 | 含义 | 说明 |
|------|------|------|
| `L:+xxxxx R:+xxxxx` | 左右编码器实时累计脉冲 | `encoder_get_counts()`，上电/清零以来的总数 |
| `<< TURN 1 REV >>` | 提示转轮子 1 圈 | 当前脉冲为零时的操作提示 |
| `PPR L:xxxxx R:xxxxx` | 当前实时脉冲（PPR 候选值） | 转轮子过程中的实时脉冲数 |
| `LOCK: L:xxxxx R:xxxxx` | 锁定的 PPR 值 | 按 KEY_EXT 锁定的最近一次测量 |
| `M1/M2/M3` | 多次测量记录 | 最多保存 3 次，求平均显示 |
| `AVG: L:xxxx R:xxxx (NX)` | N 次测量的平均值 | 用于最终确定 PPR 值 |

操作说明：

- **KEY_START（模式键）短按**：清零计数器，开始新一轮测量。
- **KEY_EXT（外接键）短按**：锁定当前计数值为 PPR（存入历史记录并自动清零准备下一次测量）。
- 流程：清零 → 手动转轮子 1 圈 → 按 KEY_EXT 锁定 → 重复 2~3 次 → 取平均值填入 `vehicle_state.h` 中的 `VEHICLE_ENCODER_PPR`。

> 若编码器安装在电机轴（减速前），每转一圈轮子的脉冲数 = 码盘孔数 × 减速比 48。例如 20 孔码盘 → PPR ≈ 960。务必实测确认！

## 串口输出格式

DEBUG 页激活后，每隔 **500 ms** 通过串口输出两行数据（波特率 9600），方便 PC 端查看或录波。

### 第一行 — 综合运行数据

```text
[DBG:INFO] E:+12 D: 45 V:30 L:1 A:2100,1900,1800,1950 TL:+35 TR:+35 ML:+34 MR:+35 FSM:2 MOT:1 LOST:0
```

| 字段 | 说明 |
|------|------|
| `DBG:INFO` | 当前 DEBUG 子页面名称（INFO / MIC / TRACE / ACT / USONIC / MOTOR / ENCODER） |
| `E` | 循迹误差 × 100 |
| `D` | 障碍距离 cm，`--` 表示 ≥50cm（远方/无效） |
| `V` | 车速 cm/s |
| `L` | 已跑圈数 |
| `A0~A3` | **4 路** ADC，逗号分隔（AO1~AO4，对应 PA2/PA3/PA6/PA7） |
| `TL / TR` | 左右目标速度 |
| `ML / MR` | 左右当前速度 |
| `FSM` | 状态机状态：0=IDLE, 1=RUNNING, 2=AVOIDING, 3=BRAKING, 4=FINISHED |
| `MOT` | 电机控制状态：0=STOP, 1=RUN, 2=BRAKE |
| `LOST` | 丢线标志：0=正常, 1=丢线中 |

### 第二行 — 新增内部状态

```text
[DBG:INFO] EL:  1234 ER:  1235 LE:  +12 LC:0 RC:0 US: 350 O:1 I:1
```

| 字段 | 说明 |
|------|------|
| `EL / ER` | 左右编码器累计脉冲 |
| `LE` | 上一次有效循迹误差 × 100 |
| `LC` | 连续丢线计数（≥3 进入丢线状态） |
| `RC` | 寻线恢复计数（≥150 超时停车） |
| `US` | 超声波原始回波时间 μs（0xFFFF 表示超时） |
| `O` | 测距完成标志（1=完成） |
| `I` | 监听回波标志（1=等待回波中） |

## 安全说明

1. DEBUG 页通过 `ui_debug_is_active()` 开关控制；未进入 DEBUG 页时，`ui_debug_update()` 不会执行任何动作。
2. 所有 IO 自检（电机、LED、蜂鸣器）仅在 `IDLE` 或 `FINISHED` 状态有效，避免影响正常行驶。
3. 退出 DEBUG 页时会调用 `ui_debug_stop_all()`，强制关闭报警 LED、蜂鸣器并停止电机。
4. 串口打印仅在 DEBUG 页激活时输出，不会在平时运行中产生额外数据。
5. 电机标定结果**断电不保存**，需手动将 OLED 显示的 LG/RG 值写入 `motion_control.c` 重新编译。
6. 编码器清零（KEY_START 在 ENCODER 页）**仅清零 OLED 显示的实时计数**（通过 `encoder_reset_totals()`），不影响车速/里程模块的增量统计。
