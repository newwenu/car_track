# 电机补偿增益手动标定说明

由于左右两侧电机、减速箱、轮胎存在机械差异，即使给两侧 PWM 相同，小车也可能会直线跑偏。`motion_control.c` 中提供了左右轮补偿增益 `s_motor_left_gain` / `s_motor_right_gain`，用于降低较快一侧的输出，使小车能近似直线行驶。

> 本说明对应 B 方案：通过 DEBUG 页面的标定功能测出建议增益，再手动填入代码重新编译。

---

## 一、自动标定法（推荐）

进入 DEBUG 模式下的 **MOTOR** 页面，使用 KEY_EXT 触发自动标定。

### 操作步骤

1. 小车上电，进入 DEBUG 模式（长按启动键 2 秒）。
2. 短按启动键切换到 **MOTOR** 页面。
3. 将小车放在平整、开阔的地面或赛道上，确保前方 1 米以上无障碍。
4. 短按 **KEY_EXT**（外接按键）。
5. OLED 第二屏显示 `CALIB RUN 60%`，小车以 60% 速度直行 2 秒。
6. 2 秒后自动停止，OLED 显示：

```text
L:1234 R:1187
LG:0.96 RG:1.00
KEY:REDO
```

含义：
- `L`：左轮在标定期间累计脉冲数
- `R`：右轮在标定期间累计脉冲数
- `LG`：建议的左轮增益
- `RG`：建议的右轮增益

7. 把 `LG` 和 `RG` 的值手动写入 `Src/APP/motion/motion_control.c`：

```c
static float s_motor_left_gain  = 0.96f;   /* 改成标定后的 LG */
static float s_motor_right_gain = 1.00f;   /* 改成标定后的 RG */
```

8. 重新编译、下载程序。

### 提前结束

如果地面长度不够 2 秒行程，在标定运行中再次短按 KEY_EXT，会立即停车并计算当前已采集的脉冲差。

### 重复标定

对同一辆小车建议标定 2~3 次，取稳定值。若多次结果差异较大，检查：
- 地面是否平整
- 轮胎是否打滑
- 电池电压是否稳定
- 编码器接线是否松动

## 二、手动计算法（无编码器或需验证时）

如果没有编码器，或者想验证自动标定结果，可采用自由滑行测试法。

### 操作步骤

1. 临时屏蔽循迹控制，让小车 pure 直行。
2. 在 `app_fsm.c` 的 IDLE 状态，或单独写一个测试模式，给左右轮相同 PWM：

```c
motion_set_wheels(60, 60);
```

3. 让小车在平整地面上直行 3 米。
4. 测量终点相对起点的横向偏移距离 `d`（单位：米）。

### 计算公式

```text
偏移率 = d / 3.0
```

- 若小车向**右**偏，说明**左轮偏快**：

```text
MOTOR_LEFT_GAIN  = 1.0 - 偏移率
MOTOR_RIGHT_GAIN = 1.0
```

- 若小车向**左**偏，说明**右轮偏快**：

```text
MOTOR_LEFT_GAIN  = 1.0
MOTOR_RIGHT_GAIN = 1.0 - 偏移率
```

### 示例

3 米行程后向右偏移 12 cm：

```text
偏移率 = 0.12 / 3.0 = 0.04
MOTOR_LEFT_GAIN  = 0.96
MOTOR_RIGHT_GAIN = 1.00
```

写入代码：

```c
static float s_motor_left_gain  = 0.96f;
static float s_motor_right_gain = 1.00f;
```

## 三、两种方法的关系

| 方法 | 精度 | 复杂度 | 适用场景 |
|------|------|--------|----------|
| 自动标定（编码器） | 高 | 低 | 有编码器，推荐首选 |
| 手动滑行测试 | 中 | 中 | 无编码器，或验证自动结果 |

自动标定的本质和手动法一致：都是通过比较相同 PWM 下左右轮的实际运动差异，把较快一侧的 PWM 按比例降低。编码器法更精确，因为它直接测量轮子转过的脉冲数，不受地面摩擦不均影响。

## 四、验证标定效果

修改增益并重新下载后：

1. 再次进入 MOTOR 页面触发自动标定。
2. 若标定结果接近 `LG:1.00 RG:1.00`，且左右脉冲数差异明显减小，说明补偿有效。
3. 或者再次运行手动滑行测试，3 米偏移应显著减小。

---

## 五、注意事项

1. **只降低，不提升**：建议始终把较慢一侧增益设为 1.0，只降低较快一侧。这样不会超出 PWM 上限，也更安全。
2. **增益范围**：代码内部已做限制，有效范围 0.50 ~ 1.50。若标定结果超出此范围，说明机械差异过大，应检查电机/轮胎/轴承。
3. **电池电压影响**：电池电量低时电机特性会变化，建议在比赛用电池电压下标定。
4. **地面一致性**：赛道地面与练习地面摩擦系数不同可能导致补偿值轻微变化，比赛前应在赛道上重新验证。
5. **标定后不要重复应用**：自动标定过程中会临时把增益重置为 1.0/1.0，因此即使当前代码里已经有旧补偿值，也不会影响新的标定结果。

## 六、常见问题

### Q1：标定后小车还是偏？

- 检查是否把 `LG` / `RG` 正确写入了 `motion_control.c` 并重新编译下载。
- 检查左右轮安装是否松动。
- 检查地面是否倾斜或有明显摩擦差异。
- 低速和高速下电机特性不同，建议在常用速度下标定。

### Q2：标定结果 LG=1.00, RG=1.00 但小车不偏？

说明当前这辆小车左右机械差异很小，不需要补偿。保持 1.0/1.0 即可。

### Q3：OLED 显示 L 或 R 为 0？

编码器未计数，检查：
- 编码器接线
- 编码器中断是否使能
- 轮子是否真的在转（电机堵转时编码器也可能不转）
