# 循迹小车当前实现说明与调参指南

## 1. 总体架构

当前代码采用分层结构：

```
USER/main.c
    └── app.c                  // 主循环，10ms 硬件节拍，多速率调度
            ├── 1ms : TIM2 Update 中断分频产生系统 tick
            ├── 10ms: obstacle_guard_update() + motion_update() + 紧急刹车块 + mic_scan() + buzzer_update()
            ├── 20ms: trace_control_update() + fsm_update()
            ├── 50ms: ui_task()
            └── 100ms: vehicle_update() + UI 刷新
```

**节拍说明**：
- 旧版本用 `delay_ms(10)` 做固定延时，周期会随任务执行时间漂移。
- 当前版本用 **TIM2 Update 中断**（10kHz）分频产生 1ms 系统 tick，`app_update()` 等待 `s_sys_tick` 增加 10 后再进入下一轮，保证 10ms 周期精确稳定。等待期间 CPU 进入 `__WFI()` 睡眠省电。

**设计原则**：
- `app_fsm.c` 只负责状态流转，不实现具体传感器算法。
- 各功能模块独立更新，通过接口向状态机报告结果。
- 紧急刹车由**独立代码块**在 10ms 内响应，不等 fsm_update() 调度周期。

## 2. 状态机流程

状态定义在 [app_fsm.h](Src/APP/control/app_fsm.h)：

| 状态 | 含义 | 触发条件 | 退出条件 |
|------|------|----------|----------|
| `IDLE` | 待机 | 上电或复位后 | 按下 `KEY_EXT`（PB11）或拍手触发 |
| `RUNNING` | 正常循迹行驶 | 从 IDLE 启动 | 检测到障碍 / 完成 2 圈 / 规则模式到 A 点 / 99s 超时 |
| `AVOIDING` | 紧急避障 | 超声波 < 20cm（中断立即触发，主循环 10ms 内刹车） | 超声波 > 35cm 并保持 30ms |
| `BRAKING` | A 点临时停车 | 规则模式下经过 A 点 | 停车 10s 结束 |
| `FINISHED` | 完成 2 圈停车 | 跑满 2 圈或 99s 超时 | 手动复位或重新启动 |

状态流转图：

```
                      障碍<20cm
  IDLE ──启动──► RUNNING ──────────► AVOIDING
   ▲               │  │                 │
   │               │  │ 规则模式+A点      │ 障碍>35cm (30ms消抖)
   │               │  └────► BRAKING ────┘
   │               │         (10s后恢复)
   │               │
   │               └── 2圈/99s超时 ──► FINISHED (能耗制动)
   │                                       │
   └── 拍手/按键复位 ──────────────────────┘
```

**99s 运行超时保护**：若小车冲出赛道后无限运行，超过 99s 自动进入 FINISHED 状态，防止计时器溢出和电池耗尽。

## 2.1 独立紧急刹车块设计

为解决"主循环 50ms 才能刹车"的问题，现在把快速刹车和状态机解耦：

| 层级 | 文件 | 职责 |
|------|------|------|
| 中断层 | `Src/BSP/sensor/ultrasonic.c` | TIM4 捕获回波后立即计算距离，过近则调用回调设置 `emergency_request` |
| 快速循环 | `Src/APP/app.c` | 每 10ms 检查 `emergency_request`；若 RUNNING，先 `motion_brake()`，再 `fsm_handle_emergency_brake()` |
| 状态机 | `Src/APP/control/app_fsm.c` | 切到 `AVOIDING`，报警，维持刹车，等 `obstacle_guard_is_cleared()` 后恢复 |
| 释放检测 | `Src/APP/obstacle/obstacle_guard.c` | 每 10ms 更新距离，30ms 释放消抖后通知 fsm |

**标志位约定**：
- `emergency_request` 只能由超声波中断设置，由 `fsm_handle_emergency_brake()` 在切状态成功后清除。
- `s_triggered` 由 `obstacle_guard_update()` 维护，用于状态机判断"是否仍要避障"。

## 3. 各模块实现与可调参数

### 3.1 循迹控制 `trace_control.c`

**实现**：
- 读取 **4 路** ADC 循迹传感器（AO1 ~ AO4，分别对应 PA2/PA3/PA6/PA7）。AO5（PB0）保留为通用 ADC 引脚，不参与循迹计算。
- 使用**施密特触发器**判断每路黑/白状态：ADC > `TRACE_TH_HIGH_DEFAULT`（750）判定变黑，ADC < `TRACE_TH_LOW_DEFAULT`（600）判定变白，死区 150 抑制边界抖动。
- 按权重 `AO1=+1.5, AO2=+0.5, AO3=-0.5, AO4=-1.5` 计算加权平均误差（范围 -1.5 ~ +1.5）。
- **分段 PD 控制**：根据误差大小自动选择三级 P 增益：
  - 小误差（|e|<0.5）：`LINE_KP_LOW=28`，高灵敏度快速纠偏
  - 中误差（0.5~1.0）：`LINEKP_MID=20`，标准响应
  - 大误差（|e|>1.0）：`LINE_KP_HIGH=14`，防止急弯超调
- **弯道出口制动**：检测到 error 从大值快速减小时（离开弯道进入切线），施加额外反向转向量抵消惯性，防止冲线。
- **平滑速度过渡**：使用一阶低通滤波（rate=0.1），每周期向目标速度靠近 10%，消除直道/弯道速度切换的阶跃突变。
- 中间两个通道（AO2、AO3）任意一个检测到黑线且两侧都没有时，判定为居中直行，抑制小幅抖动。

**丢线处理**：
- 连续 3 次（60ms）全未检测到黑线 → 进入丢线状态。
- **直道短暂丢线保护**：若丢线前 |error|<0.4，允许继续直行 10 个周期（200ms），适应黑线断续/反光间隙。
- 丢线后按 `s_last_error` 方向惯性寻线，每 0.5s 自动切换一次方向避免单向旋转。
- 寻线超时 3s 后停车等待；若重新检测到黑线自动恢复循迹。
- 丢线期间基础速度 30%，转向量 10%。

**可调参数**（[trace_control.c](Src/APP/trace/trace_control.c#L6-L54)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `TRACE_TH_HIGH_DEFAULT` | 750 | 施密特上升阈值（白→黑）。现场若黑线不明显可调低至 600。 |
| `TRACE_TH_LOW_DEFAULT` | 600 | 施密特下降阈值（黑→白）。死区 150 抑制边界抖动。 |
| `BASE_SPEED_NORMAL` | 65 | 直道基础速度百分比。小车冲出去就调低，跑太慢就调高。 |
| `BASE_SPEED_CURVE` | 40 | 弯道基础速度百分比。弯道甩尾调低，过弯太慢可调高。 |
| `CURVE_ERROR_THR` | 0.95 | 弯道判定误差阈值。调小会更早减速。 |
| `LINE_KP_LOW` | 28 | 小误差区 P 增益（|e|<0.5，直道/微偏）。提高直道灵敏度。 |
| `LINEKP_MID` | 20 | 中误差区 P 增益（0.5~1.0）。标准响应。 |
| `LINE_KP_HIGH` | 14 | 大误差区 P 增益（|e|>1.0，急弯）。防止超调。 |
| `LINE_KD` | 8 | 微分增益。转弯抖动调大，高频抖动调小。 |
| `LINE_MAX_DIFF` | 35 | 最大转向量限制。防止差速过大。 |
| `SPEED_TRANSITION_RATE` | 0.1 | 速度过渡速率。越大切换越快，越小越平滑。 |
| `EXIT_BRAKE_ZONE` | 0.6 | 弯道出口制动触发区（|error|低于此值且正在减小）。 |
| `EXIT_BRAKE_COEF` | 4.0 | 弯道出口制动强度系数。 |
| `LOST_THRESHOLD` | 3 | 连续丢线次数（3×20ms=60ms）后进入丢线状态。 |
| `LOST_RECOVERY_TICKS` | 150 | 丢线寻线超时（150×20ms=3s）。 |
| `LOST_SPEED_PCT` | 30 | 丢线寻线基础速度。 |
| `LOST_STEER_PCT` | 10 | 丢线寻线转向量。 |
| `STRAIGHT_COAST_THR` | 0.4 | 直道判定阈值（|error|小于此值允许直道滑行）。 |
| `STRAIGHT_COAST_TICKS` | 10 | 直道丢线后继续直行的周期数（200ms）。 |

**调参建议**：
1. 先让小车静止，在 DEBUG→TRACE 页观察 4 路 ADC 值，确认施密特阈值合理（黑线 ADC 高于 750，白底低于 600）。
2. 若方向相反，把 `s_weights` 数组整体取反。
3. 先设 `LINE_KD=0`，从小到大调 KP 直到能稳定循迹。
4. 加入 `LINE_KD` 抑制转弯抖动，再调整分段增益。
5. 若弯道出口冲出，增大 `EXIT_BRAKE_COEF`（最大建议 8.0）。

### 3.2 避障检测 `obstacle_guard.c`

**实现**：
- 超声波中断在测距完成的瞬间，若距离 < `AVOID_DIST_CM`，立即通过回调 `ultrasonic_distance_ready_callback()` 设置紧急刹车请求。
- `app.c` 的 10ms 独立刹车块检测到请求后，先调用 `motion_brake()`，再同步状态机到 `AVOIDING`。
- `obstacle_guard_update()` 每 10ms 读取一次超声波距离，用于维持触发状态和释放消抖。
- 两个阈值之间有滞回（20cm ↔ 35cm），避免边界抖动。
- **滞回区间内允许渐进式解除**：当处于触发状态时，在 20~35cm 区间内允许 `clear_cnt` 累积，避免"无法解除"的卡死问题。
- 超声波增加了**分段式超时处理**：连续超时时分三个阶段（极短记忆→快速退出→明确远距），防止偶然丢波后避障无法解除。
- EMA 自适应滤波：障碍接近时快速响应（α=0.8），离开时较快恢复（α=0.5），稳态平滑（α=0.3）。

**可调参数**（[obstacle_guard.c](Src/APP/obstacle/obstacle_guard.c#L9-L12)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `AVOID_DIST_CM` | 20 | 触发避障距离。场地空旷可小点，怕撞就调大。 |
| `AVOID_CLEAR_CM` | 35 | 解除避障距离。必须大于 `AVOID_DIST_CM`。 |
| `AVOID_TRIGGER_DEBOUNCE` | 2 | 触发确认次数（2×10ms=20ms）。 |
| `AVOID_CLEAR_DEBOUNCE` | 3 | 解除确认次数（3×10ms=30ms）。 |

**紧急刹车时序**：

```
T=0      障碍出现
T=2~5ms  TIM4 中断完成测距，设置 emergency_request
T=<10ms  app.c 独立刹车块：motion_brake() + 状态机切 AVOIDING
T=10ms+  obstacle_guard_update 持续确认障碍，保持刹车
T=?      障碍移走后 30ms 解除，fsm 切回 RUNNING
```

**注意**：
- 超声波现在使用 **TIM4 1μs 分辨率**（PSC=71, ARR=4999），距离公式为 `(盲区250μs + 回波时间) / 58.0` cm。
- 若超声波数据为 0 或跳变严重，检查硬件接线（PA8 发射，PB6 回波）和 TIM4 中断优先级。

### 3.3 计圈 `lap_counter.c`

**实现**：
- 通过 `trace_control_is_all_black()` 判断 4 路传感器同时检测到黑线。
- 连续 `APOINT_DEBOUNCE` 次全黑认为经过 A 点。
- 刚启动时停留在 A 点不计圈；必须离开 A 点后再次全黑才计一圈。
- 跑满 `LAP_TARGET`（默认 2）圈认为完成。

**⚠️ 当前计圈功能已暂时禁用**：`lap_counter_update()` 中直接 `return`，原计圈逻辑被 `#if 0` 注释。调试阶段避免 A 点误触发影响丢线恢复测试。需要计圈时取消注释即可恢复。

**可调参数**（[lap_counter.c](Src/APP/mission/lap_counter.c#L5-L6)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `APOINT_DEBOUNCE` | 2 | A 点消抖次数。若赛道交叉线多，调大避免误计。 |
| `LAP_TARGET` | 2 | 目标圈数。 |

**注意**：当前假设 A 点是一根足够宽的横线，能让 4 个传感器同时压线。如果赛道只有窄线或十字交叉，需要改 A 点判据（例如只要求中间 2 路全黑，或用编码器里程辅助）。

### 3.4 报警控制 `alarm_control.c`

**实现**：
- 事件驱动：启动、过 A 点、避障、A 点停车、完成。
- `AVOIDING`：持续蜂鸣器（2kHz）+ 报警 LED 闪烁（500ms 周期）。
- `BRAKING`：每秒短鸣 + 每 0.5s 闪灯。
- 其他事件为一次性提示音。
- **蜂鸣器已改为非阻塞**：`buzzer_beep()` 只设置一个 10ms 倒计时，由 `app.c` 每 10ms 调用 `buzzer_update()` 驱动。
- **板载状态 LED (PA4)** 支持多种闪烁模式：常灭、常亮、慢闪(0.5Hz)、快闪(4Hz)、双闪、呼吸灯（软件模拟），由 FSM 各状态自动切换。

**可调参数**（[alarm_control.c](Src/APP/alarm/alarm_control.c#L7-L18)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `A_BEEP_FREQ` / `A_BEEP_MS` | 1200Hz / 150ms | 过 A 点提示音。 |
| `START_BEEP_FREQ` / `START_BEEP_MS` | 1000Hz / 100ms | 启动提示音。 |
| `FINISHED_BEEP_FREQ` / `FINISHED_BEEP_MS` | 1500Hz / 800ms | 完成提示音。 |
| `BRAKE_BEEP_PERIOD` | 50 | A 点停车时蜂鸣器周期（50×20ms=1s）。 |

### 3.5 运动控制 `motion_control.c`

**实现**：
- 接收 `-100~+100` 的目标轮速百分比。
- 每 10ms 按斜坡步进 2%（`MOTION_RAMP_STEP = 10*10/50 = 2`）平滑逼近目标速度。
- 支持 `STOP`（滑行停止）、`RUN`（运行）、`BRAKE`（能耗制动）。
- 刹车时 H 桥两输入同高，电机两端短接制动。
- **`motion_set_wheels()` 不再自动解除 `BRAKE` 状态**：设置目标速度与释放刹车分离，避免误调用导致意外松刹。
- **起步助力（Kick Start）**：从静止目标 0 切换到非零目标时，先给 62% 占空比持续 150ms 克服静摩擦/启动死区，之后恢复斜坡控制。
- **电机不对称补偿**：默认左轮增益 1.00、右轮增益 1.03，补偿左右机械差异。可在 DEBUG→MOTOR 页自动标定。
- **死区补偿**：非零 PWM 时映射到 `[40%, 100%]` 区间，保留小差速下的微调能力。

**可调参数**（[motion_control.c](Src/APP/motion/motion_control.c#L12-L27)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `MOTION_RAMP_STEP` | 2 | 每 10ms 速度变化 2%。起步打滑调小，响应慢调大。 |
| `MOTION_STARTUP_KICK_PCT` | 62 | 起步助力占空比。不打滑可调高。 |
| `MOTION_STARTUP_KICK_TICKS` | 15 | 起步助力持续 tick（15×10ms=150ms）。 |
| `s_motor_left_gain` | 1.00 | 左轮不对称补偿系数（范围 0.5~1.5）。 |
| `s_motor_right_gain` | 1.03 | 右轮不对称补偿系数。比左轮略大，补偿默认右偏。 |

**注意**：电机方向已在 `motor.c` 中实现。若现场小车前进/后退/左右转方向反了，修改 `motor.c` 中 `MOTOR_L_IN1/IN2` 和 `MOTOR_R_IN3/IN4` 的对应关系，或改 `motion_set_wheels` 中左右轮符号。电机死区补偿在 `motor.c` 中，`MOTOR_MIN_DUTY=40%`。

### 3.6 车速里程 `vehicle_state.c`

**实现**：
- 每 100ms 读取左右编码器脉冲和（`encoder_read()` 取走并清零增量）。
- 根据轮径 `VEHICLE_WHEEL_DIAMETER_MM=66mm` 和编码器每转脉冲 `VEHICLE_ENCODER_PPR=20` 计算速度和累计里程。
- 每脉冲里程 = π×66mm / (20PPR×2) ≈ 5.18mm/pulse（分母×2 是因为 encoder_read 返回左右轮之和）。
- 单位 cm/s 和 cm，供应 UI 显示。

**可调参数**（[vehicle_state.h](Src/APP/vehicle/vehicle_state.h#L7-L19)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `VEHICLE_WHEEL_DIAMETER_MM` | 66 | 驱动轮直径，单位 mm。对应电机参数表 6.6cm。 |
| `VEHICLE_ENCODER_PPR` | 20 | 编码器每转脉冲数（单轮）。需实测校准。 |
| `VEHICLE_SAMPLE_MS` | 100 | 采样周期，需与 `app.c` 中调用周期一致。 |

**⚠️ PPR 校准警告**：当前 `VEHICLE_ENCODER_PPR=20` 仅在编码器直接安装在输出轮上时成立。若编码器安装在电机轴（减速前），实际单轮 PPR = 电机轴 PPR × 减速比(48)。建议使用 DEBUG→ENCODER 页实测：转一圈轮子，记录脉冲数并更新此值。

### 3.7 声控起停 `mic.c`

**实现**：
- PB7 读取咪头比较器输出（LM393，有声音时输出低电平）。
- 软件状态机消抖：连续 3 次（30ms）低电平 → 判定有效声控信号。
- **双击识别**：第一次有效信号后进入 800ms 等待窗口，窗口内收到第二次有效信号 → 判定为连续两次拍手。
- **单次拍手（IDLE/FINISHED 状态）**→ 启动小车。
- **连续两次拍手（非 IDLE/FINISHED 状态）**→ 紧急停车复位。
- 功能通过 `mic_set_enabled()` 可由 DEBUG→MIC 页或 UI 开关控制，默认关闭。

**可调参数**（[mic.c](Src/BSP/sensor/mic.c#L7-L11)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `MIC_DEBOUNCE_CNT` | 3 | 消抖次数（3×10ms=30ms）。调大抗干扰强但响应慢。 |
| `MIC_DOUBLE_TAP_MS` | 800 | 双击识别时间窗口。调小需要更快拍手。 |

### 3.8 启动键

- 当前使用 **外接按键 `KEY_EXT`（PB11）** 作为启动/复位键。
- **板载按键 `KEY_START`（PA5）** 用于 OLED 界面切换（短按：主界面→模式设定→DEBUG，长按 2s：进入/退出 DEBUG）。
- 如果现场没有外接 PB11 按键，需要把 `app_fsm.c` 中 `key_ext_scan()` 替换为 `key_start_scan()`，并修改 UI 的按键逻辑。

## 5. 已知局限与建议改进

### 5.1 A 点判据过于简单
- **问题**：用 4 路全黑作为 A 点。若赛道有十字交叉或直角，可能误计圈。
- **当前状态**：计圈功能已暂时禁用（`#if 0`），便于调试阶段专注循迹和丢线恢复。
- **改进方向**：
  - 只要求中间 2 路全黑。
  - 用编码器里程辅助：只有在预计到达 A 点的大致里程范围内，全黑才计圈。

### 5.2 丢线恢复有基础但可增强
- **现状**：已实现直道短暂丢线滑行、丢线惯性寻线（带相位切换）、寻线超时停车。
- **改进方向**：超时后增加倒车/后退寻线逻辑，避免弯道外侧冲出赛道后无法回到线上。

### 5.3 主循环节拍已稳定
- **已改进**：用 **TIM2 Update 中断**（10kHz/10=1kHz）产生系统 tick，`app_update()` 等待 `s_sys_tick` 增加 10 后进入下一轮。周期严格 10ms，计时准确，且等待期间 CPU 进入 `__WFI()` 睡眠省电。

### 5.4 启动键与 UI 键分离
- **现状**：PB11 外接按键启动，PA5 板载按键管理 UI。避免冲突。
- **改进方向**：如果只想用板载 KEY_START，可改为短按切 UI 模式、长按启动小车。

### 5.5 声控已完整实现
- **现状**：`mic.c` 已完成，支持单次拍手启动和连续两次拍手急停。通过 `mic_set_enabled()` 可由 DEBUG→MIC 页开关。
- **注意**：需先在 DEBUG→MIC 页按 KEY_START 启用声控，默认关闭。

### 5.6 编码器 PPR 需实测标定
- **问题**：`VEHICLE_ENCODER_PPR=20` 是理论值（码盘 20 孔/转，编码器在输出轮上）。若编码器在电机轴（减速前），实际 PPR 需乘减速比 48。
- **改进方向**：使用 DEBUG→ENCODER 页实测：转一圈轮子读脉冲数，更新 `vehicle_state.h` 中的 `VEHICLE_ENCODER_PPR`。

### 5.7 计圈暂时禁用
- **问题**：调试阶段为避免 A 点误触发影响丢线恢复测试，计圈功能已 `#if 0`。
- **恢复方法**：取消 [lap_counter.c](Src/APP/mission/lap_counter.c#L29-L57) 中 `return` 和 `#if 0` 注释。

### 5.8 异形循迹（十字/直角）未充分验证
- **现状**：分段 PD + 弯道出口制动 + 平滑速度过渡可处理中小角度弯道。直角弯和十字交叉可能仍需前瞻预判或分段策略。

## 6. 现场调试步骤建议

1. **静态检查传感器**：
   - 进入 DEBUG→TRACE 页，把小车放在黑线上观察 4 路 ADC 值，确认施密特阈值 750/600 合理。
   - 把小车左右移动，确认 `trace_control_get_error()` 变化方向正确（偏右时 error>0）。
   - 确认 4 路 ADC 基线接近（差异在硬件容差范围）。

2. **单独测试电机**：
   - 进入 DEBUG→MOTOR 页，短按 KEY_START 调节速度，确认前进方向正确。
   - 若方向反，修改 `motor.c` 或 `motion_set_wheels`。

3. **电机补偿标定**：
   - 在 DEBUG→MOTOR 页，短按 KEY_EXT 触发自动标定。
   - 小车以 60% 速度直行 2 秒，自动计算建议增益。
   - 将结果写入 `motion_control.c` 中 `s_motor_left_gain` / `s_motor_right_gain`。

4. **编码器 PPR 实测**：
   - 进入 DEBUG→ENCODER 页，按 KEY_START 清零。
   - 手动转一圈轮子，按 KEY_EXT 锁定 PPR 值。
   - 多次测量取平均，更新 `vehicle_state.h`。

5. **声控测试**：
   - 进入 DEBUG→MIC 页，按 KEY_START 启用声控（蜂鸣器 1200Hz 短鸣确认）。
   - 拍手观察 TRG 计数是否增加。单次拍手启动，两次拍手急停。

6. **低速跑直线**：
   - 把 `BASE_SPEED_NORMAL` 设低（如 40），`LINE_KP_LOW` 设小（如 15），`LINE_KD=0`。
   - 看小车能否基本沿直线走，再逐步加大参数。

7. **加入弯道和避障**：
   - 调通直道后，加入弯道速度参数和分段 KP。
   - 用手靠近超声波，确认小车刹车并报警，移开后 30ms 恢复。

8. **调 A 点停车和完成停车**：
   - 取消 `lap_counter.c` 中的 `#if 0` 注释恢复计圈。
   - 在规则模式下跑圈，确认每圈到 A 点精确停车 10s。
   - 确认跑满 2 圈后进入 `FINISHED`，能耗制动停车，计时停止。
