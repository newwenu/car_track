# 循迹小车当前实现说明与调参指南

## 1. 总体架构

当前代码采用分层结构：

```
USER/main.c
    └── app.c                  // 主循环，10ms 硬件节拍，多速率调度
            ├── 1ms : TIM2_CH3 输出比较中断，系统 tick
            ├── 10ms: obstacle_guard_update() + motion_update() + 紧急刹车块 + buzzer_update()
            ├── 20ms: trace_control_update() + fsm_update()
            ├── 50ms: ui_task()
            └── 100ms: vehicle_update() + UI 刷新
```

**节拍说明**：
- 旧版本用 `delay_ms(10)` 做固定延时，周期会随任务执行时间漂移。
- 当前版本用 **TIM2_CH3 输出比较中断**产生 1ms 系统 tick，`app_update()` 等待 `s_sys_tick` 增加 10 后再进入下一轮，保证 10ms 周期精确稳定。

**设计原则**：
- `app_fsm.c` 只负责状态流转，不实现具体传感器算法。
- 各功能模块独立更新，通过接口向状态机报告结果。
- 紧急刹车由**独立代码块**在 10ms 内响应，不等 fsm_update() 调度周期。

## 2. 状态机流程

状态定义在 [app_fsm.h](Src/APP/control/app_fsm.h)：

| 状态 | 含义 | 触发条件 | 退出条件 |
|------|------|----------|----------|
| `IDLE` | 待机 | 上电或复位后 | 按下 `KEY_EXT`（PB11） |
| `RUNNING` | 正常循迹行驶 | 从 IDLE 启动 | 检测到障碍 / 完成 2 圈 / 规则模式到 A 点 |
| `AVOIDING` | 紧急避障 | 超声波 < 20cm（中断立即触发，主循环 10ms 内刹车） | 超声波 > 35cm 并保持 50ms |
| `BRAKING` | A 点临时停车 | 规则模式下经过 A 点 | 停车 10s 结束 |
| `FINISHED` | 完成 2 圈停车 | 跑满 2 圈 | 手动复位或重新启动 |

状态流转图：

```
IDLE ──KEY_EXT──► RUNNING ──A点×2──► FINISHED
   ▲                 │
   │                 ├── 障碍<20cm ──► AVOIDING ──障碍>35cm──┐
   │                 │                                      │
   │                 └── 规则模式+A点 ──► BRAKING(10s)───────┘
   └──── FINISHED 可再次启动 ────────────────────────────────┘
```

## 2.1 独立紧急刹车块设计

为解决“主循环 50ms 才能刹车”的问题，现在把快速刹车和状态机解耦：

| 层级 | 文件 | 职责 |
|------|------|------|
| 中断层 | `Src/BSP/sensor/ultrasonic.c` | TIM4 捕获回波后立即计算距离，过近则调用回调设置 `emergency_request` |
| 快速循环 | `Src/APP/app.c` | 每 10ms 检查 `emergency_request`；若 RUNNING，先 `motion_brake()`，再 `fsm_handle_emergency_brake()` |
| 状态机 | `Src/APP/control/app_fsm.c` | 切到 `AVOIDING`，报警，维持刹车，等 `obstacle_guard_is_cleared()` 后恢复 |
| 释放检测 | `Src/APP/obstacle/obstacle_guard.c` | 每 10ms 更新距离，50ms 释放消抖后通知 fsm |

**标志位约定**：
- `emergency_request` 只能由超声波中断设置，由 `fsm_handle_emergency_brake()` 在切状态成功后清除。
- `s_triggered` 由 `obstacle_guard_update()` 维护，用于状态机判断“是否仍要避障”。

## 3. 各模块实现与可调参数

### 3.1 循迹控制 `trace_control.c`

**实现**：
- 读取 5 路 ADC 循迹传感器（AO1 ~ AO5）。
- 对每一路判断 `ADC > TRACE_BLACK_THR` 认为在黑线上。
- 按权重 `AO1=+2, AO2=+1, AO3=0, AO4=-1, AO5=-2` 计算加权平均误差。
- 用 PD 控制输出左右轮目标速度百分比：
  - `diff = LINE_KP * error + LINE_KD * (error - last_error)`
  - `left = base_speed + diff`
  - `right = base_speed - diff`
- 当 `|error| > CURVE_ERROR_THR` 时降低基础速度到 `BASE_SPEED_CURVE`。

**可调参数**（[trace_control.c](Src/APP/trace/trace_control.c#L7-L14)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `TRACE_BLACK_THR` | 2000 | 黑线判断阈值。现场若黑线不明显，适当调低；若环境光干扰大，适当调高。 |
| `BASE_SPEED_NORMAL` | 55 | 直道基础速度百分比。小车冲出去就调低，跑太慢就调高。 |
| `BASE_SPEED_CURVE` | 30 | 弯道基础速度百分比。弯道甩尾调低，过弯太慢可调高。 |
| `CURVE_ERROR_THR` | 1.0 | 弯道判定误差阈值。调小会更早减速，调大会更晚减速。 |
| `LINE_KP` | 18 | 比例增益。舵偏/跑偏调大，震荡调小。 |
| `LINE_KD` | 5 | 微分增益。trace 周期改为 20ms 后，从 12 按比例下调。转弯抖动调大，高频抖动调小。 |

**调参建议**：
1. 先让小车静止，打印或观察 `trace_control_get_error()`，确认黑线在中间时 error≈0，黑线偏右时 error>0，偏左时 error<0。
2. 若方向相反，把 `s_weights` 数组整体取反。
3. 先设 `LINE_KD=0`，从小到大调 `LINE_KP` 直到能稳定循迹。
4. 加入 `LINE_KD` 抑制转弯抖动。

**注意**：
- 当前 ADC 阈值是经验值，必须根据现场赛道和传感器高度实测调整。
- `trace_control_get_error()` 已修复，现在返回的是当前真实误差（之前版本返回 0，仅影响调试观察）。

### 3.2 避障检测 `obstacle_guard.c`

**实现**：
- 超声波中断 `TIM4_IRQHandler` 在测距完成的瞬间，若距离 < `AVOID_DIST_CM`，立即通过回调 `ultrasonic_distance_ready_callback()` 设置紧急刹车请求。
- `app.c` 的 10ms 独立刹车块检测到请求后，先调用 `motion_brake()`，再同步状态机到 `AVOIDING`。
- `obstacle_guard_update()` 每 10ms 读取一次超声波距离，用于维持触发状态和释放消抖。
- 两个阈值之间有滞回，避免在边界抖动。
- `ultrasonic_get_distance()` 增加了**丢波超时恢复**：若 30ms 内未收到回波，自动标记距离无效并重新触发下一次测量，避免偶然丢波后测距卡死。

**可调参数**（[obstacle_guard.c](Src/APP/obstacle/obstacle_guard.c#L5-L8)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `AVOID_DIST_CM` | 20 | 触发避障距离。场地空旷可小点，怕撞就调大。 |
| `AVOID_CLEAR_CM` | 35 | 解除避障距离。必须大于 `AVOID_DIST_CM`。 |
| `AVOID_TRIGGER_DEBOUNCE` | 2 | 触发确认次数（2×10ms=20ms）。 |
| `AVOID_CLEAR_DEBOUNCE` | 5 | 解除确认次数（5×10ms=50ms）。 |

**紧急刹车时序**：

```
T=0      障碍出现
T=2~5ms  TIM4 中断完成测距，设置 emergency_request
T=<10ms  app.c 独立刹车块：motion_brake() + 状态机切 AVOIDING
T=10ms+  obstacle_guard_update 持续确认障碍，保持刹车
T=?      障碍移走后 50ms 解除，fsm 切回 RUNNING
```

**注意**：
- 超声波距离系数已修正为 `1.7 cm/tick`（对应 TIM4 10kHz 计数），原代码 `0.017` 得到的是米，会导致阈值失效。
- 若超声波数据为 0 或跳变严重，检查硬件接线（PA8 发射，PB6 回波）和 TIM4 中断优先级。

### 3.3 计圈 `lap_counter.c`

**实现**：
- 通过 `trace_control_is_all_black()` 判断 5 路传感器同时检测到黑线。
- 连续 `APOINT_DEBOUNCE` 次全黑认为经过 A 点。
- 刚启动时停留在 A 点不计圈；必须离开 A 点后再次全黑才计一圈。
- 跑满 `LAP_TARGET`（默认 2）圈认为完成。

**可调参数**（[lap_counter.c](Src/APP/mission/lap_counter.c#L5-L6)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `APOINT_DEBOUNCE` | 2 | A 点消抖次数。若赛道交叉线多，调大避免误计。 |
| `LAP_TARGET` | 2 | 目标圈数。 |

**注意**：当前假设 A 点是一根足够宽的横线，能让 5 个传感器同时压线。如果赛道只有窄线或十字交叉，需要改 A 点判据（例如只要求中间 3 路全黑，或用编码器里程辅助）。

### 3.4 报警控制 `alarm_control.c`

**实现**：
- 事件驱动：启动、过 A 点、避障、A 点停车、完成。
- `AVOIDING`：持续蜂鸣器 + 报警 LED。
- `BRAKING`：每秒短鸣 + LED 闪烁。
- 其他事件为一次性提示音。
- **蜂鸣器已改为非阻塞**：`buzzer_beep()` 只设置一个 10ms 倒计时，由 `app.c` 每 10ms 调用 `buzzer_update()` 驱动。这样 800ms 完成提示音不会阻塞主循环，A 点停车 10s 也不会被蜂鸣器延时拉长。

**可调参数**（[alarm_control.c](Src/APP/alarm/alarm_control.c#L6-L16)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `A_BEEP_FREQ` / `A_BEEP_MS` | 1200Hz / 150ms | 过 A 点提示音。 |
| `START_BEEP_FREQ` / `START_BEEP_MS` | 1000Hz / 100ms | 启动提示音。 |
| `FINISHED_BEEP_FREQ` / `FINISHED_BEEP_MS` | 1500Hz / 800ms | 完成提示音。 |
| `BRAKE_BEEP_PERIOD` | 50 | A 点停车时蜂鸣器周期（50×20ms=1s）。 |

### 3.5 运动控制 `motion_control.c`

**实现**：
- 接收 `-100~+100` 的目标轮速百分比。
- 每 10ms 按斜坡步进 `MOTION_RAMP_STEP=1` 平滑逼近目标速度（保持与原来 50ms/5% 相同的总斜坡时间约 1s）。
- 支持 `STOP`（滑行停止）、`RUN`（运行）、`BRAKE`（能耗制动）。
- 刹车时 H 桥两输入同高，电机两端短接制动。
- **`motion_set_wheels()` 不再自动解除 `BRAKE` 状态**：设置目标速度与释放刹车分离，避免误调用导致意外松刹。从 `AVOIDING`/`BRAKING` 恢复 `RUNNING` 时，由状态机显式调用 `motion_stop()` 释放刹车后再下发新目标。
- **`FINISHED` 状态使用能耗制动**（`motion_brake()`），避免到终点后惯性冲线。

**可调参数**（[motion_control.c](Src/APP/motion/motion_control.c#L8-L10)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `MOTION_RAMP_STEP` | 1 | 每 10ms 速度变化 1%。起步打滑调小，响应慢调大。 |

**注意**：电机方向已在 `motor.c` 中实现。若现场小车前进/后退/左右转方向反了，修改 `motor.c` 中 `MOTOR_L_IN1/IN2` 和 `MOTOR_R_IN3/IN4` 的对应关系，或改 `motion_set_wheels` 中左右轮符号。

---

### 3.6 车速里程 `vehicle_state.c`

**实现**：
- 每 100ms 读取左右编码器脉冲和。
- 根据轮径 `VEHICLE_WHEEL_DIAMETER_MM=65mm` 和编码器每转脉冲 `VEHICLE_ENCODER_PPR=20` 计算速度和累计里程。
- **里程单位已修正**：之前版本把 mm 当 cm 累加，导致 UI 显示值是实际里程的 10 倍；现在 `s_distance_cm` 正确按 cm 累计。

**可调参数**（[vehicle_state.h](Src/APP/vehicle/vehicle_state.h#L7-L11)）：

| 参数 | 默认值 | 作用 |
|------|--------|------|
| `VEHICLE_WHEEL_DIAMETER_MM` | 65 | 驱动轮直径，单位 mm。 |
| `VEHICLE_ENCODER_PPR` | 20 | 编码器每转脉冲数。 |
| `VEHICLE_SAMPLE_MS` | 100 | 采样周期，需与 `app.c` 中调用周期一致。 |

---

### 3.7 启动键

- 当前使用 **外接按键 `KEY_EXT`（PB11）** 作为启动/复位键。
- 原因：`KEY_START`（PA5，板载 USER 键）已被 UI 用于切换页面/模式，避免冲突。
- 如果现场没有外接 PB11 按键，需要把 `app_fsm.c` 中 `key_ext_scan()` 替换为 `key_start_scan()`，并禁用或修改 UI 的按键功能。

## 5. 已知局限与建议改进

### 5.1 A 点判据过于简单
- **问题**：用 5 路全黑作为 A 点。若赛道有十字交叉或直角，可能误计圈。
- **改进方向**：
  - 只要求中间 3 路全黑。
  - 用编码器里程辅助：只有在预计到达 A 点的大致里程范围内，全黑才计圈。
  - 用更宽的起跑线，并只在规则赛道使用。

### 5.2 没有丢线处理
- **问题**：小车完全偏离黑线时，`trace_control.c` 会沿用上一次 error，可能原地转圈或冲出赛道。
- **改进方向**：
  - 增加丢线计数，丢线超过一定时间进入减速/停车/寻线状态。
  - 根据历史 error 方向做惯性寻线。

### 5.3 主循环节拍已由 `delay_ms()` 改为硬件 tick
- **原问题**：`app_update()` 末尾用 `delay_ms(APP_TICK_MS)` 空转 10ms，实际周期 = 10ms + 任务执行时间，导致 UI 计时器漂移。
- **已改进**：现在用 **TIM2_CH3 输出比较中断**产生 1ms 系统 tick，`app_update()` 等待 `s_sys_tick` 增加 10 后再进入下一轮。周期严格 10ms，计时准确，且等待期间 CPU 进入 `__WFI()` 睡眠。

### 5.4 启动键冲突
- **问题**：使用 PB11 外接按键，不是板载按键。
- **改进方向**：如果只想用板载 KEY_START，可改为短按切 UI 模式、长按启动小车。

### 5.5 声控未接入
- **问题**：`config.h` 中已有 `BSP_MIC`（PB7）宏，但没有 `mic.c` 驱动。
- **改进方向**：加一个 `mic.c` 读取 PB7 电平，在 `FSM_STATE_IDLE` 中检测声音触发 `fsm_start()`。

### 5.6 调试手段不足
- **问题**：没有串口输出当前 error、状态、距离等关键信息。
- **改进方向**：在 `app_update()` 中每隔 500ms 通过 `printf` 输出 `fsm_get_state()`、`trace_control_get_error()`、`obstacle_guard_get_distance()`、`lap_counter_get_laps()`，便于现场调试。

## 6. 现场调试步骤建议

1. **静态检查传感器**：
   - 把小车放在黑线上，串口打印 5 路 ADC 值，确认 `TRACE_BLACK_THR` 合理。
   - 把小车左右移动，确认 `trace_control_get_error()` 变化方向正确。

2. **单独测试电机**：
   - 临时写测试代码，调用 `motion_set_speed(30)`，确认前进方向正确。
   - 若方向反，修改 `motor.c` 或 `motion_set_wheels`。

3. **低速跑直线**：
   - 把 `BASE_SPEED_NORMAL` 设低（如 30），`LINE_KP` 设小（如 10），`LINE_KD=0`。
   - 看小车能否基本沿直线走，再逐步加大参数。

4. **加入弯道和避障**：
   - 调通直道后，加入弯道速度参数。
   - 用手靠近超声波，确认小车刹车并报警，移开后恢复。

5. **调 A 点停车**：
   - 在规则模式下跑圈，确认每圈到 A 点**精确停车 10s**，蜂鸣器 LED 提示正常。
   - 若误计圈，调整 `APOINT_DEBOUNCE` 或 A 点判据。

6. **调完成停车**：
   - 确认跑满 2 圈后进入 `FINISHED`，执行**能耗制动**停车，计时停止。
   - 若冲过终点线，检查 `motion_brake()` 是否生效或弯道速度是否过快。
